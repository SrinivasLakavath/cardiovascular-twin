# FINAL ARCHITECTURE

**Explainable Data-Driven Cardiovascular Digital Twin
(Physiology-Inspired, Surrogate-Based)**

---

## 0. One-line definition (memorise this)

> *A physiology-inspired simulator is used to generate controlled cardiovascular responses, from which an explainable machine-learning surrogate digital twin is learned for patient-specific what-if analysis.*

This line anchors everything.

---

## 1. High-level system view (conceptual)

```
Patient Profile + Intervention
            ↓
Physiology-Inspired Core (Windkessel)
            ↓
Synthetic Cardiovascular Responses
            ↓
ML Surrogate Digital Twin
            ↓
Explainable AI (XAI)
            ↓
What-If / Decision Support Analysis
```

Important:

* **The ML model is the digital twin**
* The physics core is only a *teacher*

---

## 2. Layer-by-layer architecture (this is the meat)

---

## Layer 1: Patient Profile Generator

(**State Definition**)

This defines *who* the twin represents.

### Inputs

* Age
* Sex (optional, categorical)
* Baseline SBP
* Baseline DBP
* Baseline HR
* Risk group (optional abstraction)

### Purpose (be clear)

* Not real patients
* Defines **initial state space**
* Controls variability

This prevents the criticism:

> “Your twin has no patient individuality”

---

## Layer 2: Physiology-Inspired Core

(**Canonical Cardiovascular Model**)

### What it is

* Minimal lumped-parameter Windkessel model
* Single governing ODE
* Deterministic

### What it produces

* Blood pressure trajectory
* SBP / DBP values

### What it is *not*

* ❌ Not a clinical simulator
* ❌ Not a full heart model
* ❌ Not claimed to be accurate

### How you justify it

> “We use a canonical lumped-parameter cardiovascular abstraction to capture first-order hemodynamic trends.”

This layer gives you **academic legitimacy** without fragility.

---

## Layer 3: Intervention Abstraction Layer

(**Drug → Parameter Mapping**)

This is where your architecture becomes *smart*.

Instead of simulating drugs directly:

| Intervention | Model Parameter Change      |
| ------------ | --------------------------- |
| Beta-blocker | ↓ Cardiac output (Q)        |
| Vasodilator  | ↓ Peripheral resistance (R) |
| Stimulant    | ↑ Q                         |
| Volume load  | ↑ Compliance (C)            |

Why this is good:

* Explicit assumptions
* Interpretable
* XAI-friendly
* Defensible

You are modelling **effects**, not molecules.

---

## Layer 4: Synthetic Data Factory

(**Controlled Experiment Engine**)

This is where *engineering discipline* shows.

For each synthetic patient:

1. Generate baseline parameters
2. Apply intervention mapping
3. Run physiology core
4. Record:

   * Inputs
   * ΔSBP, ΔDBP, ΔHR

### Output dataset

A clean tabular dataset like:

```
[Age, Baseline_SBP, Baseline_DBP, HR,
 Drug_Class, Dosage,
 ΔSBP, ΔDBP, ΔHR]
```

This is **gold** for ML + XAI.

---

## Layer 5: ML Surrogate Digital Twin

(**The Actual Twin**)

This is the **heart of the project**.

### Model type

* MLP / Gradient Boosting / Random Forest
* Regression (multi-output)

### Why ML is justified

* Fast inference
* Patient-specific behavior
* Enables real-time what-if queries
* Abstracts simulator complexity

### Formal definition

> The trained ML model represents a **surrogate digital twin**, approximating the cardiovascular response function for a given patient-intervention pair.

This wording matters.

---

## Layer 6: Explainable AI (XAI)

(**Trust & Transparency Layer**)

You don’t “add SHAP”.
You **design for explanation**.

### XAI objectives

* Identify dominant drivers of response
* Validate physiological plausibility
* Compare patients
* Compare interventions

### Outputs

* Global feature importance
* Local (patient-specific) explanations
* Dose–response sensitivity plots

This is where you separate yourself from generic ML projects.

---

## Layer 7: What-If & Scenario Analysis

(**Twin Interaction Layer**)

This proves it’s a *digital twin*, not a model.

Examples:

* Same patient → different drugs
* Same drug → different dosages
* Different patients → same intervention

You show:

* Counterfactual reasoning
* Scenario comparison
* Personalized response trends

This is **non-negotiable** for DT claims.
